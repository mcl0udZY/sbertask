## Используемая метрика

Для оценки качества используется метрика `Hit@10`.

Она показывает, как часто настоящий следующий товар попадает в список из 10 рекомендаций.

Формула:

```text
Hit@10 = количество успешных попаданий / количество всех тестовых сессий
```

Если реальный товар есть в топ-10 рекомендаций, это считается попаданием.

## Основная идея алгоритма

В проекте используется гибридный подход, который объединяет несколько простых стратегий.

### 1. Популярные товары

Считается, какие товары чаще всего встречаются во всех сессиях.

Эти товары используются для дополнения списка рекомендаций, если основная модель не смогла набрать 10 товаров.

### 2. Переходы между товарами

Система анализирует, какие товары чаще всего идут друг за другом.

Например, если после товара `10` часто смотрят товар `25`, то товар `25` будет хорошей рекомендацией после товара `10`.

### 3. Ассоциации внутри одной сессии

Если товары часто встречаются в одной и той же сессии, между ними создаётся связь.

Это помогает рекомендовать товары не только по последнему просмотренному товару, но и на основе всей истории пользователя в текущей сессии.

## Как работает программа

Программа выполняет следующие шаги:

1. Загружает сессии из файла `sessions.jsonl`.
2. Удаляет слишком короткие сессии, если их длина меньше 3.
3. Делит каждую сессию на:

   * обучающую часть;
   * тестовую цель.
4. Считает популярность товаров.
5. Считает переходы между соседними товарами.
6. Считает ассоциации между товарами внутри одной сессии.
7. Для каждой сессии формирует топ-10 рекомендаций.
8. Считает качество рекомендаций по метрике `Hit@10`.


## Описание основных функций

### `train_test_split(sessions)`

Делит каждую сессию на обучающую часть и тестовую цель.

```python
train_sessions = [session[:-1] for session in sessions]
test_targets = [session[-1] for session in sessions]
```

Например:

```python
[1, 5, 8, 12]
```

превращается в:

```python
train = [1, 5, 8]
target = 12
```

---

### `hit_at_k(recommendations, true_items, k)`

Считает метрику `Hit@K`.

Если правильный товар находится среди первых `k` рекомендаций, засчитывается попадание.

---

### `count_popular_items(sessions)`

Считает, сколько раз каждый товар встречался во всех сессиях.

Эта информация используется, чтобы дополнить рекомендации популярными товарами, если основных рекомендаций оказалось меньше 10.

---

### `count_transitions(sessions)`

Считает переходы между соседними товарами.

Например, для сессии:

```python
[1, 2, 3]
```

будут учтены переходы:

```text
1 → 2
2 → 3
```

Если такой переход встречается часто, то следующий товар получает больший вес в рекомендациях.

---

### `count_associations(sessions)`

Считает связи между товарами, которые встречались в одной сессии.

Например, если товары `5`, `8` и `10` были в одной сессии, то между ними создаются ассоциации.

Такие связи помогают учитывать не только последний товар, но и всю историю просмотров пользователя.

---

### `make_recommendations(train_sessions, k=10)`

Основная функция, которая строит рекомендации.

Для каждой сессии она:

1. Берёт последний просмотренный товар.
2. Добавляет товары, которые часто идут после него.
3. Добавляет товары, связанные с товарами из текущей сессии.
4. Если рекомендаций меньше 10, дополняет список популярными товарами.
5. Возвращает список рекомендаций для каждой сессии.


## Итог

В результате была реализована рекомендательная система, которая строит топ-10 рекомендаций на основе пользовательских сессий.

Алгоритм учитывает:

* популярность товаров;
* переходы между соседними товарами;
* ассоциации товаров внутри одной сессии.

Такой подход является простым, но достаточно эффективным.
